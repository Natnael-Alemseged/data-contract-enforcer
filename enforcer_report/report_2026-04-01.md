## Phase 1 — Contract Generator

**File:** `contracts/generator.py`

The ContractGenerator profiles any JSONL data file and emits:
- A Bitol v3.0.0 YAML contract with schema clauses, quality checks, and lineage metadata
- A dbt `schema.yml` with structural tests (not_null, accepted_values, unique)
- A timestamped schema snapshot in `schema_snapshots/{contract_id}/`

### Key Clause Decisions

- **Confidence fields** → constrained to `minimum: 0.0, maximum: 1.0` (catches the silent ×100 scale bug)
- **`_id` fields** → `format: uuid, pattern: ^[0-9a-f-]{36}$`
- **`_at` / `_time` fields** → `format: date-time`
- **Low-cardinality string fields** → `enum:` list detected automatically
- **Ambiguous columns** → annotated via Claude-3-Haiku on OpenRouter (LangSmith traced)

### LangSmith Integration

The `annotate_with_llm` function is instrumented with `@traceable(name="contract-llm-annotation", run_type="llm")`. Traces are sent to LangSmith project `pr-worthwhile-attitude-40`. The full `generate()` pipeline is wrapped as a `contract-generator` chain.

## Phase 2A — Validation Runner

**File:** `contracts/runner.py`

Executes every contract clause against a live data snapshot. Never crashes — malformed records produce `ERROR` status, not exceptions.

### Check Types

| Check | Description |
|-------|-------------|
| `required` | Column must exist; records must be non-null |
| `type` | dtype matches contract (number/integer/string/boolean) |
| `enum` | Values in accepted set |
| `uuid_pattern` | Matches `^[0-9a-f-]{36}$` |
| `datetime_format` | Parseable via `datetime.fromisoformat()` |
| `range` | Numeric min/max within contract bounds |
| `statistical_drift` | Z-score against baseline; WARN >2σ, FAIL >3σ |

## Phase 2B — Violation Attributor

**File:** `contracts/attributor.py`

When a check fails, the Attributor:
1. Traverses the lineage graph upstream (reverse BFS) to find source files
2. Runs `git log --follow --since="90 days ago"` per upstream file (or reads `.cartography/git_history.txt` as fallback)
3. Builds a ranked blame chain (confidence = `1.0 − (days × 0.1) − (hops × 0.2)`)
4. Computes blast radius via forward BFS to count downstream affected nodes
